# 第 0 阶段决策记录（历史）

## 已确定

- 报名优先于“做完产品”：当前提交物是开题报告。
- 核心服务先独立于飞书，确保可测试、可复现、可容灾。
- 飞书将承担机器人入口、互动卡片、多维表格留痕与人工升级，不成为第二套材料数据库。
- 首版数据必须以审核状态区分；未审核数据不能对外做材料结论。

## 原定下一道门槛

在进入真实推荐前，需要至少完成：

1. 6 类代表材料的官方来源登记；
2. 7 条规则的可验证字段；
3. 3 个演示场景的人工期望结果；
4. 飞书赛事租户/应用权限可用。

## 2026-07-17 实施更新

- 本地规则、API、React 界面、12 条官方来源材料和 30 场景评测已经完成工程化升级。
- 飞书 Aily、机器人和多维表格适配器已实现；真实联调仍取决于赛事租户和用户持有的应用凭据。

## 2026-07-18 叙事校准

- 技术方案、规则边界和验收门槛不变；对外叙事从“用户选材工具”提升为“面向 Polymaker 的可审核决策与反馈资产”。
- 正面对标官方 Web App 的既有能力，不把它误写成只有个人问答；PolyPilot 重点验证反事实解释、版本审计和飞书组织协作增量。
- 飞书不再被描述为结果展示渠道，而是 Aily 解析、Card 2.0 交互、多维表格反馈与复盘组成的产品机制。
- 业务收益和 30–60 分钟人工基线必须通过外部试点验证，不能由内部回归测试替代。
