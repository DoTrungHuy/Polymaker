# PolyPilot：面向 3D 打印场景的可解释 AI 选材顾问

> 用途：作为飞书「2026 AI 先锋未来人才大赛」Polymaker 命题的报名和开题底稿。提交前请把所有 `[待填]` 项替换为真实信息；当前版本不宣称已接入飞书或已采用 Polymaker 官方产品参数。

## 一句话项目介绍

PolyPilot 将用户“想打印什么”的自然语言需求，结合打印机能力和使用环境，转化为可追溯的材料候选、打印前提、排除理由与人工升级建议，帮助初学者避免因“材料适配错误”造成反复试错。

## 报名短文案（约 110 字）

PolyPilot 是一个面向 3D 打印用户的 AI 选材顾问。它不直接让大模型猜材料，而是先追问会改变结论的关键信息，再依据结构化材料知识、设备约束和可解释规则输出候选方案、打印前提、排除理由及风险提示。项目将以飞书机器人、互动卡片和多维表格构建“对话—决策—留痕—反馈”的协作闭环。

## 一、问题与机会

### 1. 痛点

3D 打印选材并不是“PLA、PETG、TPU 哪个更好”的单选题。真实决策同时受到零件用途、柔韧性、耐热、户外暴露、打印机喷嘴和热床上限、封箱条件、用户经验等约束影响。新手常见的失败路径是：先按模糊印象选料，之后才发现设备无法打印、使用环境不匹配，或虽然能打印但缺乏安全余量。

### 2. 为什么现有信息不够用

产品页、参数表和经验帖往往分散在不同位置，并且不能自动回答“这台打印机、这个用途、这个新手能不能用”。单纯聊天式回答还会带来两个问题：不清楚建议根据什么得出；当信息缺失或风险高时，模型仍可能给出看似确定的结论。

### 3. 核心机会

把“材料知识”做成可追溯的数据，把“设备与用途限制”做成硬规则，再把自然语言理解交给 AI。这样既能保留对话入口的易用性，也能让每一次推荐有证据、能复盘、可逐步迭代。

## 二、解决方案

### 1. 核心闭环

```text
用户描述需求
  → AI 提取初始约束
  → 追问缺失的关键条件
  → 规则过滤不可行材料
  → 候选排序与解释
  → 飞书卡片展示结果/前提/排除理由
  → 写入多维表格留痕
  → 用户反馈用于优化规则与知识库
```

### 2. 五种明确的决策状态

| 状态 | 含义 | 系统行为 |
| --- | --- | --- |
| `NEED_MORE_INFO` | 关键参数缺失 | 只问一条最能改变结论的问题，不强行推荐 |
| `RECOMMEND` | 通过约束且证据充分 | 输出候选、理由、参数与来源 |
| `CONDITIONAL` | 可以尝试但有前提/余量不足 | 高亮前提，禁止被表述为“无条件推荐” |
| `NO_COMPATIBLE_MATERIAL` | 当前条件下没有可靠候选 | 给出排除原因与可调整路径 |
| `REFUSE_OR_ESCALATE` | 安全关键或证据不足 | 不给确定结论，转人工复核 |

### 3. 飞书内的产品形态（MVP）

| 飞书载体 | MVP 中的作用 |
| --- | --- |
| 飞书机器人 / AI 对话入口 | 接收“我想打印什么”，完成关键追问 |
| 互动卡片 | 展示前三候选、适配分、前提条件、排除原因和“重新提问”按钮 |
| 多维表格 | 记录材料条目、来源、推荐日志、用户反馈和规则版本 |
| 飞书文档 / 知识空间 | 承载审核通过的材料来源摘要、术语解释与案例复盘 |

> 接入方式将以赛事提供的飞书租户、AI 能力与开发文档为准；在获得权限前，先用本地 API 验证完整决策链路。

## 三、创新点

1. **从“聊天推荐”改为“受约束的决策”。** AI 负责理解意图和组织语言，规则负责边界与排除，避免直接生成看似专业却不可验证的结论。
2. **把不确定性产品化。** 系统不是只有“推荐/不推荐”，而是让用户清楚看见需要补充的信息、推荐前提和人工复核入口。
3. **证据可追溯。** 每个正式参数必须绑定官方页面或技术资料、抓取日期、适用条件和审核状态；来源未审核时不能作为正式推荐依据。
4. **飞书协作原生。** 推荐记录、反馈、规则版本和待核验数据在飞书内协作沉淀，形成可持续运营的材料知识资产。

## 四、首版可验证 MVP

### 范围控制

首版只验证 6 类代表性材料场景和 7 条关键规则，不追求“覆盖几百种材料”。目标是证明系统能稳定完成“追问—过滤—解释—留痕”的闭环。

| 验证维度 | MVP 要证明的事情 |
| --- | --- |
| 模糊需求 | 能将口语化需求拆为用途、柔韧性、耐热、户外等字段 |
| 设备限制 | 能因喷嘴、热床、封箱、耐磨喷嘴等条件排除候选 |
| 不确定性 | 缺关键信息时先追问，而不是直接给材料名 |
| 可解释性 | 每条候选显示依据、前提与排除理由 |
| 高风险边界 | 对安全关键或证据不足场景转人工复核 |
| 飞书闭环 | 对话、卡片结果、日志与反馈在飞书中可追溯 |

### 三个演示用例

1. **柔性线缆导向件**：需要柔韧、低耐热，普通开放式打印机。预期：系统筛出柔性候选，并提示参数仍需以审核后的官方资料为准。
2. **户外支架**：户外暴露但未给出最高温度。预期：系统先追问使用温度，而不是直接推荐。
3. **高温结构件 + 入门级设备**：耐热要求高、设备能力不足。预期：系统给出“暂无可靠候选”，说明限制来自哪里，并建议升级设备或人工复核。

## 五、技术路线

```text
自然语言输入
  → 意图/约束抽取（飞书 AI 或大模型）
  → 结构化请求
  → 数据源检索（材料条目 + 来源状态）
  → 硬约束过滤（设备、用途、风险）
  → 可解释排序
  → 卡片化输出
  → 推荐日志与反馈写入多维表格
```

当前已完成的本地底座：

- 5 种决策状态与 7 类规则的可运行决策核心；
- FastAPI 接口：健康检查、材料列表、推荐接口；
- 6 条**演示用途**材料夹具，全部标识为 `demo_only`；
- 数据来源登记模板，避免真实数据未经审核进入正式推荐；
- 8 项自动化测试，覆盖追问、设备限制、高风险拒答、边界条件和 API 返回。

> 以上“已完成”仅指本地开发底座，不等同于飞书已接入，也不等同于已具备 Polymaker 官方材料参数。

## 六、数据与安全治理

1. 真实材料数据先登记来源，再由人工审核后才能标记为 `approved`。
2. 不把未知、冲突或缺失参数默认当作“满足”。
3. 将医疗、承重安全、食品接触等高风险用途转人工复核，不输出确定性结论。
4. 保留推荐版本、规则命中、输入条件和来源状态，便于复盘。
5. 不公开再分发受版权保护的官方技术文档全文，只保存链接、摘要和结构化字段。

## 九、答辩时的 45 秒表达

“我们做的不是另一个材料聊天机器人。PolyPilot 先把用户的模糊描述转成可验证约束，再用设备和用途的硬规则筛掉不可能的候选，最后把推荐前提、排除原因和数据来源状态一起给出。它在飞书里完成对话、卡片化决策、协作留痕和反馈闭环。首版不虚报覆盖几百种材料，而是用少量有来源的代表性数据，证明系统在信息不足、高风险和设备不匹配时知道该追问、该拒答、该升级给人工。”
